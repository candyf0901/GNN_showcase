## 二、实验A：评估 LLM 预测结果（evaluate.py）✅ 已复现

### 有什么用？

论文说"LLM 做任务规划不行"，你得先证明这一点。`evaluate.py` 就是用来验证 LLM 的 Direct 推理到底有多差。

### 代码流程

```
读取 prediction/ 下的预计算结果 → 读取 data/ 下的真实标签
→ 计算 F1、Accuracy、Hallucination → 打印表格
```

### 输入

- `prediction/huggingface/CodeLlama-13b/direct.json`：LLM 对 497 个测试样本的预测结果
- `data/huggingface/data.json`：每个样本的真实答案（Ground Truth）

### 输出

```
| Dataset   | LLM         | Mode | NF    | LF    | Acc   | NH-1  | NH-2  | LH-1  | LH-2  |
|-----------|-------------|------|-------|-------|-------|-------|-------|-------|-------|
| huggingface | CodeLlama-13b | chain | 0.5755 | 0.2888 | 0.1429 | 0.1656 | 0.4306 | 0.4228 | 0.6338 |
```

### 关键指标含义

| 指标 | 全称 | 含义 | 正常范围 |
|------|------|------|---------|
| **NF (Node-F1)** | 节点 F1 | 预测的工具和真实工具的匹配度 | 0.5-0.9 |
| **LF (Link-F1)** | 边 F1 | 预测的工具间连接和真实连接的匹配度 | 0.2-0.5 |
| **Acc** | Accuracy | 整条任务链完全正确的比例 | 0.1-0.3 |
| **NH-1** | Node Hallucination-1 | 幻觉出的不存在工具的比例（按次） | 0.05-0.2 |
| **NH-2** | Node Hallucination-2 | 幻觉出的不存在工具的比例（按样本） | 0.1-0.5 |
| **LH-1/2** | Link Hallucination | 同上，但针对边的幻觉 | 0.2-0.6 |

### 在你这台电脑上怎么跑的

```bash
cd GNN4TaskPlan
.venv\Scripts\python evaluate.py --dataset=huggingface --llm=CodeLlama-13b --method=direct
```

实际上就是读取已经存在的 `prediction/huggingface/CodeLlama-13b/direct.json` 文件，和 `data/huggingface/data.json` 对比，输出评估结果。**不需要 GPU，不需要模型，秒出结果。**

## 三、实验B：SGC 训练无关方法（trainfree/sgc.py）✅ 已复现

### 有什么用？

这就是论文的核心方法——**把 GNN 加进来后到底能提升多少？**

SGC 是"Simple Graph Convolution"的缩写，简单说就是：
- 把每个工具的文本描述转成向量（用 e5 模型）
- 在工具关系图上做"邻居信息传播"
- 用传播后的向量去做工具匹配

### 代码流程

```
加载 e5-large 模型（335M 参数）
       ↓
加载工具描述 → 转成向量（每个工具得到一个向量）
       ↓
加载工具关系图 → SGC 图传播（邻居信息融合）
       ↓
加载 LLM 预测的步骤 → 把每个步骤转成向量
       ↓
贪心算法：对每个步骤，从上一个工具的邻居中选择最匹配的工具
       ↓
计算 Node-F1、Link-F1、Accuracy
       ↓
对比 Direct vs SGC（不同 α 值）
```

### 核心：SGC 图传播的数学原理

```
h' = α · h + (1−α) · A · h

h = 工具编码向量
A = 归一化的邻接矩阵（描述工具间的连接关系）
α = 融合参数（0.5~1.0）

α=1.0 → 完全不使用图信息（等于 No Graph）
α=0.5 → 自己信息占50%，邻居信息占50%
α=0.8 → 自己信息占80%，邻居信息占20%（最佳值）
```

### 你在电脑上看到的结果

以 HuggingFace + CodeLlama-13B 为例：

```
| Alpha   | Node-F1 | Link-F1 | Accuracy |
|---------|---------|---------|----------|
| Direct  | 0.5755  | 0.2888  | 0.1429   | ← 纯 LLM，没有 SGC
| 0.50    | 0.6096  | 0.3410  | 0.1590   |
| 0.65    | 0.6483  | 0.3869  | 0.2133   |
| 0.80    | 0.6580  | 0.4021  | 0.2354   | ← 最佳！
| 1.00    | 0.6338  | 0.3744  | 0.2093   | ← No Graph（无图传播）
```

### 这张表告诉我们什么？

1. **Direct 最差** ↓：LLM 直接推理效果最差
2. **α=0.8 最佳** 🏆：加 20% 的邻居信息效果最好
3. **No Graph 也比 Direct 好** 🤔：这是因为 SGC 用的是 e5-large（专用句子编码器）做匹配，比让 LLM 自己选工具更靠谱。但加了图传播（α=0.8）比不加强（α=1.0）
4. **Link-F1 提升最大** 🎯：图传播对"工具间依赖关系"的改善最明显（+39%）

### 对比论文原文的结果

论文原文 Table 1 中的 CodeLlama-13B + HuggingFace 数据：

| 方法 | 论文原文的 Node-F1 | 我们复现的 Node-F1 | 吻合？ |
|------|-------------------|-------------------|--------|
| Direct | 57.55 | 57.55 | ✅ 完全一致 |
| SGC | 65.51 | 65.80 | ✅ 基本一致 |

> ⚠️ 为什么有点小差异？论文中 SGC 用的是最佳 α，我们跑的结果用的是 α=0.8（即输出中的 65.80），论文中的 65.51 可能是 α=0.75 的结果。不同 α 值结果略有不同，这是正常的。

## 四、实验C：跑 LLM 推理（trainfree/direct.py）❌ 未复现

### 为什么没做？

这个实验需要**部署一个真正的 LLM**（如 CodeLlama-13B），这在个人电脑上基本不可能：
- CodeLlama-13B 需要至少 26GB 显存
- 需要安装 vllm 或 FastChat 等推理框架
- 推理 500 个测试样本需要数小时

### 好在论文作者已经提供了预计算结果

在 `prediction/` 目录下，已经有 CodeLlama-13B、Mistral-7B、GPT-3.5-turbo、GPT-4-turbo 等多个 LLM 在多个数据集上的预测结果。
所以我们直接用这些结果做后续评估，**等价于已经做了这个实验**。

## 五、实验D：训练 GNN（traingnn/main.py）❌ 未复现

### 为什么不跑？

这个实验需要：
1. 已经运行过实验C（有 LLM 预测结果）
2. 在 HuggingFace/Multimedia/DailyLife 数据集上训练 GNN
3. 需要 GPU 训练约 15 分钟

### 效果预览

论文说训练后的 GraphSAGE 比 SGC 还要好：

| 方法 | Node-F1 | Link-F1 | Accuracy |
|------|---------|---------|----------|
| Direct | 57.55 | 28.88 | 14.29 |
| SGC (训练无关) | 65.51 | 39.44 | — |
| **GraphSAGE (训练)** | **67.30** | **42.41** | **26.56** |

### 要跑的话：

```bash
cd GNN4TaskPlan\traingnn
python main.py --dataset=huggingface --gnn_name=GraphSAGE
```

大约 15 分钟训练完，但因为我们没跑 LLM 推理（实验C），需要先用实验C的结果做数据准备。
