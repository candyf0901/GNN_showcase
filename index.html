1 概述 · Overview

该论文探讨了一个核心问题：LLM 在 Agent 任务规划（Tool Planning）中表现不佳， 能否通过图学习（Graph Learning）来提升性能？

核心发现：LLM 直接预测工具调用序列时，准确率仅 14-25%，且容易"幻觉"（Hallucination）出不存在或不合理的工具连接。 通过在工具关系图（Tool Graph）上做图传播（SGC），可以显著修正错误的规划结果。

2 问题 · LLM 规划为什么这么难？

Agent 任务规划要求 LLM 从数十个候选工具中选择正确的工具并连接成合理的调用链。 论文实验发现，即便使用 CodeLlama-13B、Mistral-7B 这样的模型：

LLM 经常：

  • 选择不存在的工具（Node Hallucination）
  • 工具间连接错误或不存在（Link Hallucination）
  • 遗漏关键工具
3 方法 · 如何用图学习改善规划

论文提出 SGC（Simple Graph Convolution） 方法， 在工具关系图上对工具编码进行图传播，然后重新排序选择最合理的工具链。

LLM 犯错分析： LLM 选择了 Text-to-Speech（语音合成）， 但用户实际需要的是音频降噪（Audio-to-Audio 工具）。 在工具关系图中，Text-to-Speech 与 Audio-to-Audio 有相似特征， 但图传播让模型更倾向于选择图中更合理的邻居。

6 α 参数分析 · How Much Graph?

SGC 的 α 参数控制原始编码与图传播的融合比例：α 越小，图信息占比越大。 实验发现最佳 α 值在 0.75~0.85 之间。

α 值Node-F1Link-F1Accuracy标注
Direct0.57550.28880.1429基线
0.500.60960.34100.1590
0.650.64830.38690.2133
0.800.65800.40210.2354🏆 最佳
1.00 (No Graph)0.63380.37440.2093

注意 α=1.0（No Graph）仍然比 Direct 好——这是因为 SGC 的编码器（e5-large） 比 LLM 的任务编码质量更高，即使没有图传播也有提升。加了图传播后进一步提升。

💬 为什么选这篇论文？

选这篇不是因为它是热门方向，而是我觉得它的切入角度很有意思。大家都在用 prompt engineering 让 LLM 做规划， 但这篇论文换了一个思路——它不从 LLM 本身下手，而是问了一个问题： LLM 不擅长规划，是不是因为它天生就不适合处理"图"这种结构？ 然后用理论证明了确实如此，再提出用 GNN 来弥补这个缺陷。我觉得这种"找到根因再对症下药"的思路很漂亮。

🔬 复现后发现：

实际跑下来发现，效果比我想象中还要明显。LLM 的 Link-F1 只有 28.88%，加了一层简单的 SGC 图传播就直接拉到 40.21%， 提升了将近 40%。而且这个方法完全不需要训练，不增加推理成本，直接拿来用就能提升。 这说明图结构信息和 LLM 的语言理解能力确实是互补的—— LLM 理解"用户要什么"，GNN 理解"工具之间怎么连"，各司其职。

🚀 未来可以继续做的方向：

这次复现的是训练无关的 SGC 版本。论文里还提到训练后的 GraphSAGE 效果更好，但需要标注数据。 我觉得之后可以往少样本或弱监督的方向继续探索——如何在只有少量标注数据的情况下， 也能训练出有效的图增强规划模型。另外，论文的实验主要集中在 HuggingFace 这类 API 调用场景， 这个方法能不能推广到更开放的真实世界任务规划（比如机器人控制、工作流自动化），也是值得继续跟进的方向。
